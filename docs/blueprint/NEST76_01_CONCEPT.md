# NEST76 — Concept

---

# Product Philosophy

NEST does not try to change how people behave.

Instead, it observes existing human interactions and records them.

The goal is to capture **trust events**.

---

# Human Behavior Model

Common real-world situation:

Driver arrives
Target person not present
Item handed to someone else

Without documentation, confusion happens.

Examples:

“Laundry saya sudah sampai belum?”
“Dititip ke siapa?”

NEST provides digital proof.

---

# Product Behavior

The system should feel like a **small intelligent tool**.

Users should be able to operate it instantly without instructions.

---

# System Tone

All messages must be friendly and empathetic.

Example message:

Sepertinya nomor HP Budi tidak benar.

Kami tidak bisa mengirim notifikasi ke sana.

Tapi tenang, kamu tetap bisa kirim linknya secara manual kok.

The system must **never blame the user**.

---

# Product Identity

NEST should feel:

human
simple
trustworthy

It should not feel:

corporate
technical
complicated
